Objective

Ensure that every product image displayed across the KalaKart UI:

Matches the exact product category and type

Looks authentic, handcrafted, and realistic

Remains consistent across all pages (preview, listing, and detail view)

📦 1. STRICT IMAGE–PRODUCT MAPPING RULE (CRITICAL)

For every product card and detail page:

The image MUST:

Represent the exact product type

Belong to the correct category

Example Rules:

If category = Paintings / Drawings
→ Show actual artwork (canvas, sketches, paintings)
❌ NOT pottery, ❌ NOT random decor

If category = Pottery / Ceramics
→ Show clay pots, vases, handmade ceramics

If category = Wood Crafts
→ Show wooden carved items, furniture, artifacts

👉 Figma AI must never reuse unrelated placeholder images

🧩 2. IMAGE CONSISTENCY ACROSS SCREENS

Ensure SAME product uses SAME image everywhere:

Pages to sync:

Product Card (listing page)

Product Preview (hover / quick view)

Product Detail Page

❌ DO NOT:

Show one image in listing and a different one in detail

✅ ALWAYS:

Maintain one consistent primary image per product

🎨 3. IMAGE STYLE GUIDELINES (VERY IMPORTANT)

Figma AI must generate/select images with:

Authenticity

Real handcrafted feel

Slight imperfections (not overly polished)

Natural textures visible

Lighting

Soft natural lighting

No harsh artificial studio look

Background

Neutral or Indian cultural context

Example:

Wooden table

Clay background

Artisan workspace

🇮🇳 4. CULTURAL ALIGNMENT (KALAKART CORE IDENTITY)

Images must reflect:

Indian traditional craftsmanship

Handmade, local artisan vibe

Examples:

Madhubani paintings → traditional patterns

Pottery → मिट्टी texture visible

Handloom → fabric texture visible

🧱 5. CATEGORY-WISE IMAGE RULESET
🎨 Paintings / Drawings

Canvas art, sketches, framed artwork

Close-up texture visible

🏺 Pottery

Clay pots, diyas, vases

Earthy tones, matte finish

🪵 Wooden Crafts

Carved wood items

Visible grain texture

🧵 Textiles

Sarees, handwoven fabrics

Folded or draped presentation

🔁 6. AUTO-REPLACEMENT LOGIC (FOR FIGMA AI)

If an image does NOT match product:

Automatically replace it with:

Correct category image

Relevant product-specific visual

📱 7. UI INTEGRATION RULES
Product Cards

Image must be:

Centered

Properly cropped (no cut-off product)

Product Detail Page

Show:

Main image (same as preview)

Optional additional angles

⚠️ 8. COMMON FAILURES TO AVOID

❌ Random stock images

❌ Same image used for multiple different products

❌ Overly aesthetic but irrelevant visuals

❌ Mismatch between category and image

❌ Changing image between pages

✅ FINAL EXPECTED RESULT

User experience should feel like:

User sees product → image matches expectation → clicks → same product continues → builds trust

NOT:

User sees product → clicks → different image → confusion → loss of trust
🚀 OPTIONAL (BUT HIGH IMPACT)

Add “real artisan photo” tag

Add multiple angles for authenticity

Add zoom-in texture view
